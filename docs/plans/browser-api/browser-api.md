# BrowserApi — Master Plan

**Plan structure:** This project follows the [large project plan structure](../../explanations/large-project-plan-structure.md) (from MitWare conventions).

## Vision

Generate typed C# wrappers for every browser API from W3C/WHATWG specs. The core library has zero dependencies — it's pure types and string serialization. Separate packages add JS interop and Blazor integration.

## Cross-Cutting Decisions

### Naming

- C# PascalCase everywhere, even when WebIDL uses camelCase
- Generated types are `partial class` to allow hand-written extensions
- Generated files have `// <auto-generated/>` header

### Type Mapping

Standard WebIDL → C# mapping (see [CLAUDE.md](../../../CLAUDE.md) for full table). Key decisions:
- `Promise<T>` → `Task<T>` with `Async` suffix
- Union types `(A or B)` → method overloads (preferred) or `OneOf<A, B>`
- `sequence<T>` → `IReadOnlyList<T>` (not arrays, for immutability)
- DOMString enums → C# enums with `[StringValue("...")]` attribute

### Serialization

Every type that represents a web value implements `ICssValue` or `IWebIdlSerializable`:
- `ToCss()` for CSS values → CSS string
- `ToJs()` for JS interop → value suitable for `IJSRuntime`

### Backend Abstraction

The interop layer is behind an interface so it can be swapped:
- Today: `IJSRuntime` (Microsoft.JSInterop)
- Future: native WASM imports via Component Model
